# 算法API如何开发

最早我写的是单脚本程序。后来做企业项目时，有时会在处理数据时用Jupyter等交互式方式捣鼓数据，有时会写若干脚本，将数据处理分析的过程划分为若干步。但这些和我现在做的都有些区别。

现在做前后端结构，开发后端API时，需要考虑前端传入什么信息，后端传回什么信息。 与大部分系统不同，我们做的是特殊的算法模块，不但无状态，甚至无数据库访问功能。可以认为是一些纯函数，一次性接受输入，再返回所有输出。（或者拆分为若干步骤，每步也是如此。）

这就产生一些后果。

- 不要考虑具体的当前业务需要什么输出。要考虑这一计算过程都产生了哪些有意义的结果，都应该输出。

类似于lisp返回mutliple value。


- 与业务相关的内容应该放在一个额外的业务模块里？
