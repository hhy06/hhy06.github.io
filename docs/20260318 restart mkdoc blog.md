# test how to create a new blog entry

It has been a while and I forgot the process. Somehow, after a little tweaking, I managed to make this run again:

1. write the markdown file in docs;

2. git add and git commit

3. git push 

Wait for github actions to take over the next step. After 10 minutes or so, it is updated.

Now I have two blogs, the old one with gridea and the new one with mkdocs.

About the order of the blog: it is actually sorted by the name of the original .md files.
